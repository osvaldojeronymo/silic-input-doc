# Plano de Entrega por Ator - SILIC 2.0

Este documento organiza o envio do prototipo de baixa fidelidade e da documentacao para tres publicos:

1. UX (prototipo de alta fidelidade no Figma)
2. Analista de Requisitos (formalizacao funcional)
3. Desenvolvedores (implementacao do modulo)

## Fluxo oficial de encaminhamento

1. Time atual envia pacote para UX.
2. UX produz prototipo de alta fidelidade no Figma e devolve com observacoes.
3. Time atual envia pacote consolidado (documentacao + Figma) para Analista de Requisitos.
4. Analista de Requisitos valida e abre a demanda para Desenvolvedores da fabrica.
