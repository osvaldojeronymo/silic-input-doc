# Checklist de Envio entre Atores

Use este checklist para garantir qualidade no repasse entre UX, Analista de Requisitos e Desenvolvedores.

## Etapa 1 - Envio para UX

1. Prototipo atual identificado como baixa fidelidade.
2. Historia principal anexada: `docs/HISTORIA_USUARIO_US001_CONTROLE_LOCACAO.md`.
3. Dicionario de dados anexado: `docs/DICIONARIO_DADOS_V1.md`.
4. Backlog anexado: `docs/BACKLOG_US001_SUBHISTORIAS.md`.
5. Escopo de telas e perfis confirmado.

## Etapa 2 - Recebimento do UX e envio ao Analista de Requisitos

1. Link Figma final recebido.
2. Estados de tela cobertos (vazio, carregando, erro, inconsistencia).
3. Regras de negocio chave representadas (perfil e D-180).
4. Pacote do Analista montado conforme `docs/entregaveis/PACOTE_ANALISTA_REQUISITOS.md`.

## Etapa 3 - Envio do Analista para Desenvolvedores

1. Documento de requisitos aprovado.
2. Matriz de rastreabilidade pronta.
3. Priorizacao P0/P1 confirmada.
4. Pacote tecnico montado conforme `docs/entregaveis/PACOTE_DESENVOLVEDORES.md`.
